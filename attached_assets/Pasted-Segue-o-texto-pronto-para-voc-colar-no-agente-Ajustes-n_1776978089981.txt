## 3. Ajuste nos KPIs "A Pagar" e "A Receber"

**Página afetada:**
`/dashboard`
**Envolve:** Backend + Frontend

### Novo formato de exibição:

Os KPIs devem exibir dois valores:

#### Para "A Pagar"

Formato:

`Pago / A Pagar`

Exemplo:

`2.350,00 / 5.800,00`

Onde:

* **Pago** → soma dos registros com status pago na competência atual
* **A Pagar** → soma total das contas da competência atual

#### Para "A Receber"

Formato:

`Recebido / A Receber`

Exemplo:

`1.200,00 / 4.500,00`

Onde:

* **Recebido** → soma dos registros recebidos na competência atual
* **A Receber** → soma total da competência atual

**Observação importante:**

O valor Pago/Recebido normalmente será menor que o total.

## 4. Formatação abreviada de valores na Dashboard

**Página afetada:**
`/dashboard`
**Somente frontend**

### Regra:

* Valores **abaixo de 1.000**
  → Exibir normal
  Exemplo:
  `523,21`

* Valores **a partir de 1.000**
  → Exibir abreviado com `k`

### Exemplos esperados:

* `523,21` → `523,21`
* `1.854,25` → `1.85k`
* `1.864,25` → `1.86k`
* `15.420,90` → `15.42k`

**Regras adicionais:**

* Sempre usar **2 casas decimais**
* Usar **ponto como separador decimal** no formato abreviado (`1.85k`)
* Não aplicar abreviação fora da dashboard

## Casos de borda importantes

1. Se não houver dados na competência:

* KPIs devem mostrar `0,00`
* Gráficos devem aparecer vazios, sem erro

2. Se existir valor zero:

* Mostrar normalmente (`0,00`)
* Não ocultar o card

3. Se alguma categoria não tiver valor:

* Não incluir no gráfico

4. Se o usuário clicar rapidamente em vários cards:

* A navegação deve continuar funcionando normalmente
* Sem duplicação de rotas ou erros de estado
